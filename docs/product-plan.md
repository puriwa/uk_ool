# 억울함해결사 제품 기획

작성 기준일: 2026-07-21

## 1. 제품 정의

첫 버전은 광범위한 법률상담 챗봇이 아니라 `교통사고 사건 준비 플랫폼`으로 시작한다.

> 교통사고가 발생한 순간부터 증거 정리, 과실 이의 준비, 변호사 상담까지 이어주는 사건 워크스페이스

권장 메시지는 `사실을 정리하고, 다음 행동을 놓치지 않게`다. 서비스 이름은 억울함을 공감하되 결과를 보장하는 표현은 사용하지 않는다.

### 역할 구분

- 무료 공개 기능: 사고 초기 대응, 절차 안내, 증거 체크리스트, 일반 템플릿
- 로그인 AI 기능: 사실관계 구조화, 자료 누락 확인, 관련 법령·판례와 출처 제시
- 변호사 기능: 구체적 법률자문, 대응 전략, 문서 검토, 수임
- 플랫폼 기능: 사건 관리, 상담 예약, 자료 공유, 기한과 다음 행동 관리

## 2. 경쟁 서비스와 제품 공백 가설

다음 평가는 공개된 서비스 화면을 기준으로 하며 사용자 인터뷰로 검증해야 한다.

| 서비스 | 공개된 강점 | 공략할 제품 공백 가설 |
|---|---|---|
| 로톡 | 변호사 검색, 사례, 전화·영상·방문상담 | 사고 증거와 기한을 지속 관리하는 사건 워크스페이스 |
| 로앤굿 | 구조화된 사건 등록, 변호사 제안 | 사고 직후 무료 대응과 수임 전 장기 준비 과정 |
| 스스로닷컴 | 블랙박스 제보와 교통사고 과실 분석 | 다수 변호사 탐색과 개인 사건 관리 |
| 과실비율정보포털 | 공식 과실기준, 심의사례, 진행조회 | 자연어 입력과 증거 정리부터 상담까지의 연결 |
| 법률똑똑이 | 무료 법률정보, 상담예약, 비용계산 | 교통사고 특화 자료 분석과 사건 관리 |
| 로폼·빅케이스 | 문서 자동작성 또는 전문 법률정보 검색 | 일반인의 사건 발생부터 해결 준비까지 이어지는 흐름 |

핵심 차별화 조합은 `사고 초기 가이드 + 증거 보관함 + AI 사실 정리 + 근거 표시 + 기한 관리 + 변호사 전달 패킷`이다.

## 3. 초기 사용자

### 일반 사용자

- 사고 직후 무엇을 해야 할지 모르는 사람
- 보험사가 제시한 과실비율에 동의하지 못하는 사람
- 분심위 결과 이후 소송 또는 법률자문을 고민하는 사람
- 블랙박스, 사진, 진단서, 보험자료를 정리하지 못한 사람

### 변호사

- 교통사고·손해배상·보험 분야를 취급하는 변호사
- 상담 전에 정리된 사실과 증거를 받고 싶은 변호사
- 상담 접수, 일정, 자료 공유를 한 화면에서 관리하려는 변호사 또는 로펌

### 운영 관리자

- 변호사 자격과 프로필 검증
- 템플릿·가이드·법률 출처 관리
- AI 응답과 인용 근거 감사
- 신고, 결제, 환불, 개인정보 요청 처리

## 4. 핵심 사용자 흐름

```text
상황 선택
→ 무료 초기 가이드
→ 로그인
→ 사건 생성
→ 영상·사진·문서 업로드
→ AI가 사실 후보 추출
→ 사용자가 사실 확인·수정
→ 근거가 포함된 사건 리포트
→ 사용자가 변호사를 직접 탐색·선택
→ 상담 예약 및 사건 패킷 공유
→ 이후 할 일과 기한 관리
```

### 첫 화면의 쉬운 메뉴

- 방금 사고가 났어요
- 보험사가 말한 과실비율이 억울해요
- 상대방이 대인·대물 접수를 거부해요
- 분심위 결과에 동의할 수 없어요
- 경찰 조사를 앞두고 있어요
- 합의해도 되는지 모르겠어요
- 변호사 상담이나 소송을 준비하고 싶어요

## 5. 초기 유입과 로그인 경계

### 로그인 없이 제공

- 사고 직후 10분 체크리스트
- 보험사에 확인할 질문
- 블랙박스·현장사진·CCTV 등 증거 체크리스트
- 경찰 신고, 보험 접수, 분심위, 소송의 절차 지도
- 사고 유형별 준비서류
- 합의 전 확인사항
- 일반적인 사실확인 메모와 템플릿

### 로그인 후 제공

- 사건 저장
- 블랙박스·사진·PDF·음성 업로드
- AI 사건 정리
- 기한과 할 일 알림
- 사건 패킷 생성
- 변호사 상담 예약과 자료 공유

가입 전에도 실질적인 가치를 제공한 뒤 저장과 분석 시점에 로그인을 요청한다.

## 6. 사건 준비도

승소 가능성이나 과실 확률 대신 자료의 완성도를 보여주는 `사건 준비도`를 제공한다.

- 사고 기본정보
- 영상·사진
- 보험자료
- 진료·손해자료
- 상담 질문

점수는 결과를 예측하지 않으며 누락된 준비 항목만 설명한다.

## 8. 다양한 입력과 AI 결과

### 입력

- 자연어 서술
- 쉬운 선택형 질문
- 블랙박스 영상
- 현장 사진
- 보험사 문서, 경찰 문서, 진단서, 견적서 등 PDF·이미지
- 음성 메모

### 초기 영상 처리 범위

초기 MVP에서는 과실비율 자동 판정을 구현하지 않는다.

- 원본 영상 저장
- 썸네일과 메타데이터 추출
- 사용자가 충돌·쟁점 시점 표시
- 중요 구간 캡처
- 얼굴·번호판 가림
- 영상 구간과 사건 설명 연결

### AI 리포트 구조

- 사용자가 직접 입력한 사실
- 자료에서 추출된 사실 후보
- 아직 확인되지 않은 추정
- 누락되거나 추가 확인이 필요한 자료
- 검토할 법적 쟁점
- 관련 법령·판례, 출처, 기준일
- 지금 해야 할 일
- 변호사에게 물어볼 질문

AI는 과실, 승소, 형량, 합의금을 확정하지 않는다. 근거가 부족하면 유보하고 추가 자료 또는 변호사 검토를 안내한다.

## 9. 데이터와 처리 구조

법령·판례의 우선 데이터 소스는 국가법령정보 공동활용 API 등 공식·일차 출처다. 데이터별 이용조건, 갱신주기, 원문 제공 범위를 확인한다.

```text
브라우저
→ 서버 API
→ 파일 저장과 악성파일 검사
→ OCR·음성변환·영상 전처리
→ 개인정보 마스킹
→ 법령·판례 검색
→ LLM 구조화 분석
→ 출처와 응답 형식 검증
→ 사건 리포트 표시
```

### 필수 원칙

- API 키는 서버에서만 보관한다.
- 답변마다 출처와 기준일을 표시한다.
- 원본 증거와 AI 요약을 분리 보존한다.
- 사용자 진술, 추출 사실, AI 추정을 데이터 모델에서도 구분한다.
- 변호사 공유 전 별도 동의를 받는다.
- 보관기간 선택, 다운로드, 삭제 기능을 제공한다.
- AI 제공자의 데이터 학습·보관 정책을 계약 단계에서 검토한다.

## 10. 규제상 제품 경계

출시 전 국내 변호사법과 변호사검색서비스 운영 가이드라인에 대한 전문 검토가 필요하다.

- 플랫폼의 유료 AI가 개별 사건에 구체적인 법률상담을 제공하지 않도록 역할을 제한한다.
- 상세 자문과 전략은 변호사가 제공한다.
- 개별 사건정보의 분석 결과를 변호사 검색·추천 순위에 사용하지 않는다.
- 사용자가 취급분야, 지역, 상담방식, 가능시간 등 객관적 조건을 직접 선택한다.
- 목록은 무작위·순환 등 공개된 중립 기준으로 정렬한다.
- 광고 여부와 정렬 기준을 명확히 표시한다.
- 수임액 비례 수수료와 성공보수 공유는 사용하지 않는다.
- 승소, 형량, 합의금, 과실비율을 보장하거나 확률로 광고하지 않는다.

### P1: 교통사고 클로즈드 베타

- 현재 P1 화면·기술 기준은 [P1 v2 상세 기획·기술 명세](p1-v2-spec.md)를 우선한다. [P1 1차 기획](p1-plan.md)은 변경 이력으로 보존한다.
- 비로그인 `3분 가이드`, 비로그인 규칙 기반 `간단 확인`, 로그인 후 `무료 AI 사건정리`를 서로 다른 흐름으로 제공한다.
- 홈의 `무료로 내 상황 확인하기`와 `3분 가이드 시작`은 서로 다른 라우트로 연결한다.
- 고객용 내 사건 공간에서 질문·자료·다음 행동을 정리하고, 질문 핵심어와 겹치는 로컬 예시 및 일반 법전 참고를 제공한다. 실제 판례·법령 원문 검색과 개별 사건 적용은 포함하지 않는다.
- Supabase 관리형 소셜 로그인과 더미 로그인 폴백
- 로컬 Codex CLI를 이용한 구조화 사건정리 기능 검증
- 실제 파일 업로드
- OCR·음성변환
- AI 사실관계 정리
- 공식 출처 인용
- 사용자 사실 확인
- 사건 패킷 생성
- 상담 예약
- 관리자 검수
