# P1 v2 패치노트

배포 대상: 교통사고 사건 준비 클로즈드 베타  
기준일: 2026-07-21  
상태: 구현 및 로컬 기능 검증 완료

## 이번 변경의 핵심

첫 방문자가 로그인 전에 상황을 고르고 필요한 준비 항목을 확인할 수 있게 했습니다. 사건별 AI 정리는 로그인 뒤에만 열리며, 답변 주체가 AI임을 먼저 알립니다.

## 새로 제공하는 흐름

- `3분 가이드`와 `무료로 내 상황 확인하기`를 서로 다른 화면으로 분리했습니다.
  - 3분 가이드는 여섯 교통사고 상황별 설명과 체크리스트를 제공합니다.
  - 간단 확인은 선택형 질문에 따라 일반 준비 항목을 제공합니다.
- `무료 AI 사건정리`를 추가했습니다.
  - 사고 설명, 보험사 의견, 사진·영상·PDF 첨부 목록을 입력할 수 있습니다.
  - 결과는 사용자 진술, 자료 사실 후보, 확인 필요 사항, 일반 정보, 다음 행동, 변호사 질문으로 분리해 보여 줍니다.
  - 과실비율·승소 가능성·합의금 등의 결과를 판단하거나 보장하지 않습니다.
- 첫 AI 입력 전 `답변 주체: AI (Codex)` 및 법률자문이 아니라는 안내 확인 절차를 추가했습니다.
- AI 결과 뒤에 더 확실한 확인을 위한 변호사 프로필·상담 준비 흐름을 연결했습니다.

## 로그인 및 내 사건

- 로그인 계정에 제공자별 색상·이모지 배지를 표시합니다.
- Supabase Auth 환경 변수가 있으면 Google·Kakao OAuth를 사용할 수 있도록 구성했습니다.
- 설정 전에는 개발·시연용 더미 로그인이 동작합니다. 네이버는 현재 더미 제공자입니다.
- `내 사건`, AI 사건정리, 변호사 대시보드는 로그인 뒤에만 접근할 수 있습니다.
- P1의 AI 결과 저장은 브라우저 `localStorage` 목업입니다. 실제 사건 데이터의 운영 저장소가 아닙니다.

## 운영 및 변호사 화면

- `/admin/dashboard`에 기간별 핵심 지표, 유입 퍼널, 주제 분포, AI 안전·품질, 운영 대기열을 보여 주는 목업 대시보드를 추가했습니다.
- 변호사 목록·상세 화면에 공개된 전문 분야, 경력, 사건 활동을 구조화해 표시했습니다.
- 해당 정보는 엘파인드 공개 검색 및 상세 프로필에서 2026-07-21에 사실형으로 요약한 참고 데이터입니다. 제휴·추천을 뜻하지 않으며, 실제 공개 전 당사자 확인과 최신성 검수가 필요합니다.

## 개발 환경의 Codex 연결

- Vite 개발 서버에서만 `/api/codex-draft` 엔드포인트를 제공합니다.
- 서버가 로컬 `codex exec`를 호출해 구조화 JSON을 생성합니다. 기본 모델은 `gpt-5.6-luna`이며 `UKOOL_CODEX_MODEL`로 바꿀 수 있습니다.
- 이는 보안·개인정보 보호 설계 전의 개발용 연결입니다. 운영 배포에 사용하면 안 됩니다.

## 검증 결과

- `npm run build` 성공
- 로컬 개발 서버에서 Codex 구조화 응답 생성 성공

## 알려진 제한사항

- 관리자·변호사 대시보드의 수치는 예시 집계 데이터입니다.
- 실제 인증 설정, 서버 저장, 파일 보관·삭제, 접근 제어, 상담 예약·결제는 아직 운영 연결하지 않았습니다.
- 첨부 영상·PDF는 현재 목록 관리 위주이며, 모든 파일 유형의 본문 추출을 제공하지 않습니다.
